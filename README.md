# ZeroProblems

React + Vite дашборд для анализа обращений граждан Омской области.

---

## Стек

- **React 18** + **Vite 5**
- **Tailwind CSS 3** — утилитарные классы + CSS-переменные для тем
- **Leaflet 1.9** + **react-leaflet 4** — интерактивная карта
- **osmtogeojson** — конвертация данных Overpass API в GeoJSON
- **Recharts 2** — горизонтальные bar-чарты в drilldown
- **lucide-react** — иконки

---

## Структура экранов

| Экран | Файл | Описание |
|-------|------|----------|
| Загрузка файла | `src/screens/UploadScreen.jsx` | Drag & drop Excel/CSV, кнопка «Пропустить — demo режим» |
| Прогресс анализа | `src/screens/ProgressScreen.jsx` | Анимированные шаги ~7 сек, переход на дашборд |
| Дашборд | `src/screens/DashboardScreen.jsx` | Карта + топ-10 таблица + 3 критических района |
| Детализация района | `src/screens/DrilldownScreen.jsx` | Bar-чарт, прогресс-бары, примеры обращений, скачать TXT |

Навигация управляется состоянием `screen` в `App.jsx`. Текущий экран сохраняется в `localStorage` (`omsk_pulse_ready`).

---

## Компоненты

- **`OmskMap`** — загружает границы районов с Overpass API, кешируется в `localStorage` (`omsk_boundaries_v3`). При недоступности API карта показывает только тайловый слой. Тайлы: CartoDB `light_all`. Ограничения: minZoom=6, maxBounds по Омской области.
- **`Top10Table`** — таблица районов по убыванию скора
- **`DistrictCard`** — карточка района с индикатором скора и топ-проблемой
- **`ThemeToggle`** — переключатель светлой/тёмной темы

---

## Данные

Все данные — **моковые**, хардкод в `src/data/mockData.js`:
- 30+ районов Омской области
- Поля: `id`, `name`, `score` (0–100), `coords`, `topProblem`, `problems[]`, `examples[]`, `summary`
- `analysisSteps[]` — шаги для ProgressScreen

---

## Что реально работает

- Полный UI всех 4 экранов
- Светлая / тёмная тема (CSS-переменные, `data-theme` на `<html>`)
- Карта с закраской районов по скору (цвет по порогам: зелёный→красный)
- Hover-подсветка районов + tooltip с названием и скором
- Клик по району → переход в DrilldownScreen
- Скачивание отчёта по району в `.txt`
- Кеширование границ карты в localStorage

## Что заглушено / не реализовано

- **Загрузка файла не парсится** — любой файл просто запускает фейковый прогресс
- **Анализ ненастоящий** — таймер, данные всегда из mockData.js
- **Нет бэкенда** — всё работает статично в браузере
- **Overpass API** — при отсутствии интернета границы не загрузятся (тайлы карты тоже)
